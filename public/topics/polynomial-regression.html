POLYNOMIAL REGRESSION (Deep Dive)

What is Polynomial Regression?

Polynomial regression is a type of regression analysis in which the relationship between the independent variable and the dependent variable is modeled as an n-th degree polynomial. It helps capture complex (curved) relationships that linear regression cannot fit well.

Why Polynomial Regression?

Choosing between linear regression and polynomial regression depends on the dataset. A non-linear dataset can't be fitted well with a straight line — polynomial features allow the model to capture non-linear patterns.

Intuition: Linear regression fits a straight line. Polynomial regression fits a curve by adding powers of \(x\) like \(x^2\), \(x^3\), ... \(x^n\).

Equation of Polynomial Regression

y = w₀ + w₁x + w₂x² + w₃x³ + ... + wₙxⁿ + ε

Quadratic (degree 2)

y = w₀ + w₁x + w₂x² + ε

How does Polynomial Regression Work?

Polynomial regression works similarly to linear regression, but it first transforms the input into polynomial features \((x^2, x^3, \dots)\). These transformed features are treated like multiple inputs, and a linear regressor is trained on them.

Key idea: Polynomial regression is a special case of multiple linear regression, where the extra features are derived from the original feature.

Step 1: Data Preparation

import numpy as np import pandas as pd import matplotlib.pyplot as plt from sklearn.preprocessing import PolynomialFeatures data = pd.read_csv('ice_cream_selling_data.csv') X = data.iloc[:, 0].values.reshape(-1, 1) y = data.iloc[:, 1].values

Visualize the original data points

plt.scatter(X, y, color="green") plt.title("Original Data") plt.xlabel("Temperature (C)") plt.ylabel("Ice Cream Sales (units)") plt.show()
Graph 1: Original Data (Scatter)

Create polynomial features (degree = 2)

degree = 2 poly_features = PolynomialFeatures(degree=degree) X_poly = poly_features.fit_transform(X)

Step 2: Model Training (Linear Regression on polynomial features)

from sklearn.linear_model import LinearRegression lr_model = LinearRegression() lr_model.fit(X_poly, y)

Step 3: Prediction

y_pred = lr_model.predict(X_poly) df = pd.DataFrame({'Actual Values': y, 'Predicted Values': y_pred}) print(df)

Step 4: Evaluation (R² Score)

from sklearn.metrics import r2_score r2 = r2_score(y, y_pred) print(r2) # example: 0.9321...

Step 5: Visualize the polynomial regression curve

plt.scatter(X, y, color="green") plt.plot(X, y_pred, color='red', label=f'Polynomial Regression (degree={degree})') plt.xlabel("Temperature (C)") plt.ylabel("Ice Cream Sales (units)") plt.legend() plt.title("Polynomial Regression") plt.show()
Graph 2: Best-fit Polynomial Curve (degree 2)

Predict for new temperature

X_new = np.array([[1.9929]]) X_new_poly = poly_features.transform(X_new) y_new_pred = lr_model.predict(X_new_poly) print(y_new_pred)

Applications in Deep Learning, NLP & Generative AI

Polynomial features help capture curvature and non-linear effects — this idea appears in many ML systems.

Deep Learning

  • Non-linear modeling intuition: polynomial terms mimic non-linear feature interactions (a stepping stone to neural nets).
  • Feature engineering baseline: compare polynomial regression vs neural networks on tabular data.
  • Calibration / regression heads: predicting continuous targets often ends with a linear layer; polynomial features can boost classical baselines.

NLP

  • Text scoring: polynomial interactions between numeric text features (length, frequency, readability metrics).
  • Embedding interactions: lightweight non-linear mapping by adding squared terms on selected embedding components (baseline experiments).
  • Sentiment intensity: predict a continuous sentiment score using engineered features.

Generative AI (GenAI)

  • Reward / quality scoring: score generated outputs from multiple signals with non-linear interactions.
  • Safety & calibration: model “risk vs quality” curves where effects are not linear.
  • Post-processing: fit non-linear relationships in evaluation metrics (e.g., temperature vs output quality).

Other common uses

Curve fitting Demand vs price curves Sensor calibration Physics/chemistry trends Weather vs yield
