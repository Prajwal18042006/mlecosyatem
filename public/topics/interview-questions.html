Overfitting vs Underfitting — Bias-Variance Intuition
Q2 What is overfitting in machine learning and how can it be avoided? ▼

Overfitting occurs when a model learns noise and random fluctuations in training data — giving high training accuracy but poor performance on unseen data.

  • Early Stopping — stop training when validation accuracy stops improving
  • Regularization — L1 (Lasso) or L2 (Ridge) penalties reduce model complexity
  • Cross-Validation — k-fold CV ensures the model generalizes well
  • Dropout — randomly drop neurons during neural network training
  • Simpler Models — avoid over-complex models when simpler ones explain the data

Underfitting occurs when a model is too simple to capture patterns — poor accuracy on both train and test data. Fix it by: using more complex models, adding relevant features, reducing regularization, or training longer.

L1 vs L2 Regularization — Penalty Shape
Q3 What is Regularization? ▼

Regularization adds a penalty term to the loss function to discourage the model from assigning too much importance (large weights) to specific features, helping it generalize better on unseen data.

  • L1 (Lasso) — penalty = |w|, can shrink weights to exactly zero (feature selection)
  • L2 (Ridge) — penalty = w², reduces large weights without eliminating them
  • Elastic Net — combines L1 + L2 to balance selection and weight reduction
  • Dropout — for neural networks, randomly drops neurons during training
Q4 Explain Lasso, Ridge, and Elastic Net Regularization with formulas. ▼

L1 (Lasso) — adds absolute value of weights as penalty. Shrinks some weights to zero → performs feature selection.

Lasso Loss
Loss = MSE + λ Σ|wᵢ|

L2 (Ridge) — adds squared weights as penalty. Reduces large weights but keeps all features.

Ridge Loss
Loss = MSE + λ Σwᵢ²

Elastic Net — combines both. Useful when features are correlated (avoids Lasso picking just one from a group).

Elastic Net
Loss = MSE + λ₁Σ|wᵢ| + λ₂Σwᵢ²

Key rule: Use L1 when many features are irrelevant. Use L2 when all features matter but you want to prevent overfitting. Use Elastic Net when features are correlated.

Section 02 Model Evaluation
Confusion Matrix — Classification Performance Grid
Precision · Recall · F1 — Metric Relationships
ROC Curve — True Positive Rate vs False Positive Rate
Q5–9 Model Evaluation Techniques — Train-Test Split, Confusion Matrix, Precision, Recall, AUC-ROC ▼

Train-Test Split — divide data into training (70–80%) and testing (20–30%) sets. Model trains on training set and is evaluated on unseen test set.

Confusion Matrix — a 2×2 table comparing predicted vs actual labels showing TP, TN, FP, FN. Used to derive Accuracy, Precision, Recall and F1.

Precision — TP / (TP + FP). How many predicted positives are truly positive. High precision avoids false alarms (e.g., spam detection).

Recall (Sensitivity) — TP / (TP + FN). How many actual positives are found. High recall avoids missing cases (e.g., disease detection).

F1-Score — harmonic mean of precision and recall. Best metric when classes are imbalanced.

F1 = 2 × (Precision × Recall) / (Precision + Recall)

ROC Curve — plots TPR vs FPR across thresholds. AUC = 1 → perfect; AUC = 0.5 → random guess. Higher AUC = better model discrimination.

Is accuracy always good? No — in imbalanced datasets, a model predicting the majority class always gets high accuracy but fails on the minority class. Use F1 and AUC instead.

k-Fold Cross-Validation (k=5) — Rotating Test Fold
Q11–12 What is Cross-Validation? Explain k-Fold, LOO, and Hold-Out methods. ▼

Cross-validation tests how well a model generalizes to unseen data by splitting the dataset into multiple subsets (folds), training and testing on different folds each time.

k-Fold CV — split data into k equal folds. Train on k-1 folds, test on the remaining fold, repeat k times. Final score = average of all k results.

CV_error = (1/k) × Σᵢ errorᵢ

Leave-One-Out (LOO) — special case of k-Fold where k = number of samples. Every single point acts as the test set once. Very accurate but computationally expensive for large datasets.

Hold-Out Method — simplest split (e.g., 70% train, 30% test). Fast but may produce biased results depending on the split.

Section 03 Feature Engineering & Selection
Feature Selection Methods — Three Families
Q13–16 Feature Engineering, Feature Selection, Dimensionality Reduction — explained ▼

Feature Engineering — creating, transforming or selecting relevant features from raw data. Better features → better model accuracy and generalization.

  • Feature Creation — extract "year" from date, "age" from date of birth
  • Transformation — log, square root, normalization, scaling
  • Encoding — one-hot encoding, label encoding for categorical variables
  • Selection — keep only the most relevant features

Feature Selection — removes irrelevant or redundant features using Filter, Wrapper, or Embedded methods.

Dimensionality Reduction — reduces number of features while retaining most information. Reduces overfitting, speeds up computation, enables 2D/3D visualization. Example: PCA reduces 100 features to 10 principal components capturing 95% variance.

Section 04 Data Preprocessing
Label Encoding vs One-Hot Encoding
SMOTE — Synthetic Minority Over-Sampling
Q17–23 Categorical Data, Missing Values, Outliers, Upsampling / Downsampling ▼

Categorical Data — features representing discrete categories (gender, color). Types: Nominal (no order: Red/Blue) and Ordinal (ordered: Low/Medium/High). ML models need numerical inputs so encoding is required.

Label Encoding — assigns an integer to each category. Use for ordinal data. Risk: implies false ordering for nominal data.

One-Hot Encoding — creates a binary column per category. Use for nominal data. Risk: high dimensionality for many categories.

Missing Values — handle by: (1) drop rows/columns, (2) impute with mean/median/mode, (3) forward/backward fill for time series, (4) flag with binary column, (5) prediction-based imputation.

Outliers — data points significantly different from others. Detected via IQR method (outside Q1–1.5×IQR or Q3+1.5×IQR) or Z-score (|z|>3). Handle by: removing, transforming (log), capping (Winsorization), or using robust models.

Upsampling (SMOTE) — creates synthetic minority samples by interpolating between existing points. Avoids pure duplication. Useful for imbalanced classification.

Downsampling — reduce majority class samples. Faster training but risks losing information.

Section 05 Bias, Variance & Hyperparameter Tuning
Bias-Variance Tradeoff — Target Analogy
Q25–26 Bias-Variance Tradeoff and Hyperparameter Tuning ▼

Bias — error from wrong assumptions. High bias = underfitting. Model too simple (e.g., linear model on non-linear data).

Variance — error from sensitivity to small fluctuations. High variance = overfitting. Model too complex (e.g., deep decision tree).

Total Error = Bias² + Variance + Irreducible Error

Decreasing bias usually increases variance and vice versa. The goal is to minimize total error.

Hyperparameter Tuning — finding the best set of hyperparameters (set before training, not learned from data):

  • Grid Search — tries all combinations. Exhaustive but slow for large spaces
  • Random Search — randomly samples combinations. Faster than grid search
  • Bayesian Optimization — builds probabilistic model to choose parameters intelligently. Most efficient
Section 06 Linear & Logistic Regression
Sigmoid Function — Logistic Regression Output
Q27–30 Linear Regression, Logistic Regression, Multicollinearity & VIF ▼

Linear Regression — predicts a continuous target variable using a linear relationship: y = mx + c. Assumptions: Linearity, Independence, Homoscedasticity, Normality of errors, No Multicollinearity.

Logistic Regression — despite the name, it's a classification model. The sigmoid function converts any real number into a probability between 0 and 1. A threshold (usually 0.5) then classifies output as 0 or 1.

Cluster-choice for optimal K: Elbow Method (plot WCSS vs K), Silhouette Score (how similar points are to own cluster), Gap Statistic.

Multicollinearity — two or more features are highly correlated with each other. Causes unstable coefficients, interpretation difficulty, inflated variance. Detected via correlation matrix or VIF.

VIF_i = 1 / (1 - R²ᵢ)
  • VIF = 1 → no correlation
  • 1 < VIF < 5 → moderate, acceptable
  • VIF > 5 → high multicollinearity, problematic

Fix multicollinearity by removing/combining correlated features, using PCA, or applying Ridge regression.

Section 07 Decision Trees
Decision Tree Structure — Nodes & Splits
Q31–35 Entropy, Information Gain, Pruning, ID3 vs CART ▼

Entropy — measure of impurity or randomness. If all samples belong to one class → entropy = 0 (pure). Mixed classes → high entropy.

Entropy(S) = −Σ pᵢ log₂(pᵢ)

Information Gain — reduction in entropy when splitting on a feature. Higher IG = better feature for splitting.

IG(S,A) = Entropy(S) − Σᵥ (|Sᵥ|/|S|) × Entropy(Sᵥ)

Decision Trees pick the feature with highest Information Gain at each node.

Pruning — removes branches that add little value. Pre-pruning stops early (max_depth, min_samples). Post-pruning grows full tree then removes low-impact branches. Prevents overfitting, improves generalization.

Section 08 Naive Bayes
Bayes' Theorem — Components
Q36–38 Naive Bayes — Theorem, Assumptions & Types ▼

Naive Bayes is a classification algorithm based on Bayes' theorem. "Naive" because it assumes all features are independent — a simplification that works surprisingly well in practice (spam detection, sentiment analysis).

Working: Calculate prior probability of each class → compute likelihood of features for each class → apply Bayes' theorem → assign class with highest posterior.

Assumptions: Feature independence given class label · all features contribute equally · correctly labeled training data.

Types:

  • Gaussian NB — continuous features, assumes normal distribution (height, weight)
  • Multinomial NB — discrete count data (word counts in text classification)
  • Bernoulli NB — binary features (word present/absent in document)
  • Categorical NB — categorical features (color: red/green/blue)
Section 09 K-Nearest Neighbors & Distance Metrics
KNN — Finding K Nearest Neighbors
Q39–45 KNN Working, Lazy Learning, K Value, Distance Metrics & KNN Imputer ▼

KNN Working: Choose K → calculate distance to all training points → select K nearest neighbors → majority vote (classification) or average (regression).

Lazy learning — KNN doesn't learn an explicit model during training. It stores all data and computes at prediction time. Memory-intensive but simple to implement.

Effect of K: Small K → sensitive to noise, overfitting. Large K → smooth boundaries, underfitting. Optimal K found via cross-validation.

Distance Metrics:

  • Euclidean — straight-line distance. Most common for continuous data
  • Manhattan — sum of absolute differences. Good for equal penalization across dimensions
  • Minkowski — generalization: p=1→Manhattan, p=2→Euclidean
  • Cosine Similarity — angle between vectors. Great for text/high-dimensional sparse data
  • Hamming — positions where elements differ. For binary/categorical data
  • Mahalanobis — accounts for correlations. Useful for multivariate data

KNN Imputer — fills missing values using mean/median of k nearest neighbors (based on Euclidean distance).

Section 10 Support Vector Machines (SVM)
SVM — Maximum Margin Hyperplane & Support Vectors
Q46–48 SVM Decision Boundary, Kernel Trick & Kernel Types ▼

Decision Boundary — the hyperplane that separates data points of different classes while maximizing the margin (distance between the hyperplane and the nearest support vectors).

Does SVM work only with linear data? No — for non-linear data, SVM uses the kernel trick to transform data into a higher-dimensional space where linear separation becomes possible.

Kernel Types:

  • Linear Kernel — linearly separable data or very high-dimensional feature spaces (e.g., text classification). Fast and simple
  • Polynomial Kernel — non-linear but polynomial boundaries (circular or curved)
  • RBF / Gaussian Kernel — most commonly used. Handles complex, highly non-linear data
  • Sigmoid Kernel — behaves like neural networks. Less common than RBF
Section 11 Ensemble Learning
Bagging vs Boosting — Parallel vs Sequential Training
Q49–56 Random Forest, Bootstrapping, AdaBoost, XGBoost, CatBoost ▼

Random Forest — builds multiple decision trees and combines by majority vote (classification) or average (regression). Reduces overfitting, handles large datasets, provides feature importance.

Diversity through: Bagging (random data subsets) + Feature Randomness (random subset of features at each split).

Bootstrapping — sampling with replacement to create multiple training sets. Same point can appear multiple times. Helps estimate variability and reduce variance.

AdaBoost — combines shallow trees sequentially. Each new tree focuses on samples misclassified by the previous. Final prediction = weighted majority vote. Best for simple models needing improvement.

XGBoost — optimized gradient boosting with L1+L2 regularization. Builds trees sequentially correcting previous errors. Highly efficient, parallelizable, widely used in competitions.

CatBoost — gradient boosting that handles categorical features natively (no encoding needed). Uses ordered boosting to reduce overfitting. Less tuning required.

Section 12 Clustering & Association Rules
K-Means — Iteration to Convergence
Q58–67 K-Means, Hierarchical Clustering, DBSCAN, GMM, Association Rules (Apriori, FP-Growth) ▼

K-Means — divides data into K clusters. Each cluster has a centroid (average). Goal: minimize distance of points to their centroid. Requires predefined K (use Elbow Method or Silhouette Score).

K-Means++ — smarter initialization by choosing centroids far apart. Reduces chance of poor clustering. Fuzzy C-Means — soft clustering where each point has a probability of belonging to multiple clusters.

Hierarchical Clustering — doesn't need predefined K. Builds a dendrogram. Agglomerative (bottom-up): merge closest clusters. Divisive (top-down): split large clusters. Linkage methods: Single (nearest), Complete (furthest), Average, Ward's (minimize variance).

DBSCAN — density-based. Groups closely packed points, marks low-density points as noise. Detects arbitrary shapes. Requires eps and minPts parameters. OPTICS extends DBSCAN for varying densities.

GMM — probabilistic clustering using Gaussian distributions. Soft assignments (probability per cluster). Handles elliptical clusters. Uses EM algorithm.

Association Rules — finds relationships in transactional data. Uses: Support (fraction of transactions with itemset), Confidence (likelihood of consequent given antecedent), Lift (co-occurrence vs independence).

Apriori — finds frequent itemsets iteratively using minimum support threshold. FP-Growth — faster alternative using FP-Tree compressed data structure. Fewer dataset scans.

Section 13 Dimensionality Reduction & Recommendation Systems
PCA — Projecting to Principal Components
Q68–74 PCA, NMF, LDA, t-SNE, Manifold Learning, Recommendation Systems, EM Algorithm ▼

PCA — transforms high-dimensional data to lower dimensions preserving maximum variance. Steps: standardize → covariance matrix → eigenvalues/eigenvectors → select top k principal components → transform. Sensitive to scaling.

NMF — factorizes non-negative matrix into two lower-rank non-negative matrices. Interpretable components. Used for topic modeling, image processing.

LDA (Latent Dirichlet Allocation) — probabilistic topic modeling. Each document = mixture of topics. Each topic = distribution over words.

t-SNE — non-linear reduction for visualization in 2D/3D. Preserves local neighborhoods (similar points stay close). Computationally expensive. Does not preserve global distances.

Manifold Learning techniques: Isomap (geodesic distances), LLE (local linear embedding), UMAP (local + global, faster than t-SNE), MDS (pairwise distances).

Content-Based Filtering — recommends items similar to what user liked before (item features). Suffers from cold start for new users.

Collaborative Filtering — recommends based on similar users (user-based) or similar items (item-based). Requires large interaction data.

EM Algorithm — for models with hidden/latent variables. E-step: estimate probabilities of hidden states. M-step: update parameters to maximize likelihood. Repeat until convergence. Used in GMM, missing data imputation.

Section 14 Time Series Analysis & Forecasting
Time Series Components — Decomposition
Q75–78 ARIMA, SARIMA, Exponential Smoothing & Concept Drift ▼

ARIMA(p,d,q) — for non-stationary time series. AR: uses past values. I: differencing for stationarity. MA: uses past forecast errors. Good when no strong seasonality.

SARIMA(p,d,q)(P,D,Q,m) — extends ARIMA with seasonal components. m = periods per season (e.g., 12 for monthly data with yearly seasonality). Use for electricity demand, retail sales.

Exponential Smoothing — gives more weight to recent observations. Three types:

  • SES — no trend or seasonality. Weight α controls recency
  • Holt's — adds trend component
  • Holt-Winters — handles both trend and seasonality (additive or multiplicative)

Concept Drift — statistical properties of data change over time, making trained models less accurate. Types: Sudden, Gradual, Recurring. Handle by: regular retraining, adaptive learning algorithms that update with new data.

Section 15 Reinforcement Learning
RL Agent-Environment Loop
Q-Learning vs SARSA — Off-Policy vs On-Policy
Q79–90 RL Fundamentals — MDP, Bellman, Value Iteration, Policy Gradient, Actor-Critic ▼

Reinforcement Learning — agent learns by interacting with environment. Takes actions → receives reward or penalty → adjusts behavior to maximize long-term cumulative reward.

MDP (Markov Decision Process) — mathematical framework: States (S), Actions (A), Transition Probability (P), Reward (R), Policy (π). Markov property: next state depends only on current state and action.

Optimal Policy (π*) — strategy that maximizes expected cumulative reward. No other policy achieves higher long-term reward.

Bellman Equation — recursive formula breaking long-term return into immediate reward + discounted future value.

V(s) = E[Rₜ₊₁ + γ V(Sₜ₊₁) | Sₜ = s]

Dynamic Programming — solves MDPs using Bellman equations when environment is fully known. Value Iteration: directly computes optimal value function. Policy Iteration: alternates evaluation and improvement.

Model-Free RL — learns from experience without knowing transition probabilities (Q-Learning, SARSA, DQN). Model-Based RL — uses/learns environment model for planning (Dyna-Q, MCTS). More sample-efficient but complex.

Deep Q-Learning (DQN) — neural network approximates Q-function. Uses Experience Replay (break correlation) and Target Network (stable targets). Handles large/continuous state spaces.

Policy Gradient — directly optimizes the policy π_θ(a|s) using gradient ascent. Works well in continuous action spaces. REINFORCE: Monte Carlo policy gradient, high variance. Actor-Critic: combines policy gradient (actor) + value function (critic). Lower variance, basis of A2C and PPO.
