SIMPLE LINEAR REGRESSION (Deep Dive)

What is Simple Linear Regression?

Simple linear regression is a supervised learning method where a single independent variable is used to predict a dependent variable. It models a linear relationship between variables.

Core Concept

It finds the best-fit straight line that minimizes error between actual and predicted values.

Formula

Y = w₀ + w₁X + ε

  • Y → Dependent Variable
  • X → Independent Variable
  • w₀ → Intercept
  • w₁ → Slope
  • ε → Error

How It Works

The model tries to minimize error using a loss function (MSE) and finds the optimal line.

Cost Function:

J(w₀,w₁) = (1/2n) Σ(Y - Ŷ)²

Step 1: Data Preparation

First, import the libraries and load the dataset.

import numpy as np
import matplotlib.pyplot as plt
import pandas as pd

dataset = pd.read_csv('Salary_Data.csv')
X = dataset.iloc[:, :-1].values
y = dataset.iloc[:, -1].values

Check the first five examples.

print(dataset.head())

Check if the dataset is linear

If the points follow a roughly straight pattern, simple linear regression is a good fit.

plt.scatter(X, y, color="green")
plt.title("Salary vs Experience")
plt.xlabel("Years of Experience")
plt.ylabel("Salary (INR)")
plt.show()
Graph: Salary vs Experience (dataset scatter)

Split the dataset into training and testing sets

We use 80% for training and 20% for testing (24 train, 6 test out of 30).

from sklearn.model_selection import train_test_split
X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=0.2)

Step 2: Model Training (Fitting the model)

from sklearn.linear_model import LinearRegression

regressor = LinearRegression()
regressor.fit(X_train, y_train)

Step 3: Model Testing

y_pred = regressor.predict(X_test)
df = pd.DataFrame({'Actual Values': y_test, 'Predicted Values': y_pred})
print(df)
Graph: Train/Test points + fitted regression line

Step 4: Model Evaluation

Common metrics: MSE, RMSE, MAE, and R².

from sklearn.metrics import mean_squared_error, mean_absolute_error, r2_score

y_pred = regressor.predict(X_test)
mse = mean_squared_error(y_test, y_pred)
rmse = mean_squared_error(y_test, y_pred) # (note: you can use sqrt(mse) for RMSE)
mae = mean_absolute_error(y_test, y_pred)
print("mse", mse)
print("rsme", rmse)
print("mae", mae)

Step 5: Visualize Training Set Results (with Regression Line)

y_pred_train = regressor.predict(X_train)
plt.scatter(X_train, y_train, color="green", label="training data points (actual)")
plt.scatter(X_train, y_pred_train, color="blue", label="training data points (predicted)")
plt.plot(X_train, y_pred_train, color="red")
plt.title("Salary vs Experience (Training Dataset)")
plt.xlabel("Years of Experience")
plt.ylabel("Salary (In Rupees)")
plt.legend()
plt.show()

Applications (Real-world + Deep Learning + NLP + GenAI + Agentic AI)

Simple linear regression is small, but the idea (weighted sum + bias) is everywhere.

Real-world applications

Salary prediction House price estimation Sales forecasting Fuel consumption prediction Crop yield estimation

Deep Learning connection

Neuron math: Output = \(w_1x_1 + w_2x_2 + \dots + b\)
This is the same structure as linear regression (then neural nets add non-linear activation functions).
  • Used as the final layer for regression outputs
  • MSE loss is the standard objective for many regression models
  • Gradient descent learns weights the same way (optimize error)

NLP applications

  • Sentiment score regression: predict a score like -1 to 1
  • Readability / complexity: predict a continuous difficulty score
  • Embedding → scalar: map text embeddings to numerical outputs

GenAI applications

  • Calibration: adjust scores/probabilities (a common “linear head” pattern)
  • Reward modeling (RLHF): predict a “quality” score for generated text
  • Projection layers: transform vector representations between spaces

Agentic AI applications

Agentic systems often need quick scoring functions to decide actions.

  • Tool selection scoring: score which tool is best for a user query
  • Risk scoring: estimate risk/cost before executing an action
  • Budget/time prediction: predict time or cost of a plan step
